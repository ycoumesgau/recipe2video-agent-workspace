# Scale and Proportion Guardrails

Use this checklist whenever a food object can be interpreted at the wrong size.

## 1) Lock real-world size

State one explicit numeric anchor when possible:

- diameter (`3-4 cm`, `18-20 cm`);
- height/thickness (`2 cm thick`, `6 cm tall`);
- weight class (`bite-size`, `single-portion`, `family-size`) only if numeric size is unavailable.

## 2) Add a kitchen-relative anchor

Add one relative constraint tied to known context:

- `fits in one palm`;
- `smaller than a ladle bowl`;
- `sits centered on a standard dinner plate with visible rim`;
- `occupies about one third of the kitchen island width`.

## 3) Lock count for repeated topology

When repeated elements appear, specify count or range:

- `10-14 arancini balls`;
- `8 choux domes`;
- `one ring made of separate domes, not a smooth torus`.

## 4) Add negative scale constraints

Always state what must not happen when risk exists:

- `not soccer-ball sized`;
- `not oversized compared to the character hand`;
- `not miniature marble-sized`.

## 5) Keep continuity across segments

Repeat the same size anchors in every related segment:

- copy the same numbers and relative anchors;
- mention continuity in `continuity` (`same arancini diameter as segment-02`);
- mention risk in `risk` if scale drift is likely.

## 6) Ask before guessing

If the recipe source is ambiguous (example: only says "make balls"), ask:

- target diameter range;
- expected count;
- serving format (single bite vs plated main portion).
